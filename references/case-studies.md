# Case Studies

## Case 1: AI 周报压缩时间

### User Input
帮我写一篇小红书，主题是：我怎么用 AI 把周报时间从 2 小时压到 20 分钟。

### What the skill should emphasize
- 时间节省的结果感
- 打工人强共鸣
- 方法可复制
- 不要写得像课程广告

### Ideal angle
从“以前每周最烦的是写周报”切入，用前后对比强化真实感。

## Case 2: AI 副业尝试

### User Input
请围绕“下班后用 AI 接文案私单”写一篇小红书，语气真实一点，别太鸡血。

### What the skill should emphasize
- 副业不是暴富故事，而是可执行尝试
- 突出起步门槛低
- 写清楚 AI 在哪里帮上忙
- 保持克制，不夸大收益

### Ideal angle
从“不是我突然变厉害了，是我终于不用一切都手写”切入。

## Case 3: 标题 A/B 测试

### User Input
下面是正文，帮我出 10 个小红书标题，分成结果型、痛点型、反差型三组。

### What the skill should emphasize
- 不是同义改写
- 三组风格明显不同
- 标题要像平台原生内容
- 避免老套营销词

### Ideal angle
让用户能直接拿去测试点击率，而不是只看起来“像标题”。

---

## Case 4: 热帖结构迁移

### User Input
分析这篇爆帖为什么火，然后按类似结构帮我写一篇关于 AI 做会议纪要的小红书。

### What the skill should emphasize
- 先拆逻辑，再写新稿
- 学结构，不照抄原句
- 把场景替换成新的工作流
- 保持节奏和平台感
